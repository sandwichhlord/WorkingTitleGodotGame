Basic Rules:-

1. Always pull before your coding sesh starts. run git pull origin dev before you even start up godot so you dont code on an outdated timeline
2. Branch everything, even agar its a miniboss animation ka tweak or a huge adjustment, make branches for it.
3. Dont make huge commits every night with commit messages like "made changes", make small commits every few minutes or every time you complete a small task with details of that small task
4. Commit messages should explain what you did and why it was required
5. DONT TOUCH CORE CODE WITHOUT PROPER HEAVY DISCUSSION ABOUT IT IN A MEETING FIRST
6. dont make changes in anyone elses code ofc
7. make sure everything is working great before committing
8. Never make change to any scenes (.tscn file) without first making sure no one else is working on it
